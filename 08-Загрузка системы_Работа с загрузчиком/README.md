# __8. Загрузка системы_Работа с загрузчиком __

## Включить отображение меню Grub

Для отображения меню нужно отредактировать конфигурационный файл и комментируем строку, скрывающую меню и ставим задержку для выбора пункта меню в 10 секунд.

Установим следующие пакеты:
``` bash
nano /etc/default/grub

#GRUB_TIMEOUT_STYLE=hidden
GRUB_TIMEOUT=10
```

Обновляем конфигурацию загрузчика и перезагружаемся для проверки.

``` bash
update-grub
reboot
```
при перезагрузке убеждаемся что видно меню загрузчика.


## Попасть в систему без пароля несколькими способами

Для получения доступа открываем GUI VirtualBox, запускаем VM и при выборе ядра для загрузки нажимаем e - в данном контексте edit. 
Попадаем в окно, где мы можем изменить параметры загрузки:

### Способ 1. init=/bin/bash
В конце строки, начинающейся с linux, добавляем init=/bin/bash и нажимаем сtrl-x для загрузки в систему
Рутовая файловая система при этом монтируется в режиме Read-Only. Если вы хотите перемонтировать ее в режим Read-Write, можно воспользоваться командой:
``` bash
mount -o remount,rw /
```

После чего можно убедиться, записав данные в любой файл или прочитав вывод команды:
``` bash
mount | grep root
/dev/mapper/vg_root-lv_root on / type ext4 (rw, realtime)
```

### Способ 2. Recovery mode

В меню загрузчика на первом уровне выбираем второй пункт (Advanced options…), далее загружаем пункт меню с указанием recovery mode в названии. 
Получим меню режима восстановления.

В этом меню  включаем поддержку сети (network) для того, чтобы файловая система перемонтировалась в режим read/write (либо это можно сделать вручную).
Далее выбираем пункт root и попадаем в консоль с пользователем root. Если вы ранее устанавливали пароль для пользователя root (по умолчанию его нет), то необходимо его ввести. 
В этой консоли можно производить любые манипуляции с системой.


## Установить систему с LVM, после чего переименовать VG

Первым делом посмотрим текущее состояние системы (список Volume Group):

``` bash
vgs
  VG        #PV #LV #SN Attr   VSize   VFree
  ubuntu-vg   1   2   0 wz--n- <13.25g <3.25g
  vg_root     1   1   0 wz--n- <10.00g     0
  vg_var      2   1   0 wz--n-   2.99g  1.12g
```

Приступим к переименованию:
``` bash
vgrename ubuntu-vg ubuntu-vg-otus
  Volume group "ubuntu-vg" successfully renamed to "ubuntu-vg-otus"
```

Далее правим /boot/grub/grub.cfg. Везде заменяем старое название VG на новое (в файле дефис меняется на два дефиса ubuntu--vg ubuntu--otus).
После чего можем перезагружаться и, если все сделано правильно, успешно грузимся с новым именем Volume Group и проверяем:

``` bash
vgs
  VG        #PV #LV #SN Attr   VSize   VFree
  ubuntu-vg-otus  1   2   0 wz--n- <13.25g <3.25g
  vg_root     1   1   0 wz--n- <10.00g     0
  vg_var      2   1   0 wz--n-   2.99g  1.12g
```
При желании можно так же заменить название Logical Volume.
